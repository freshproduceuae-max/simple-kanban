# Operating Model

Goal: decide how the project AI team should operate before the rest of the playbook runs.

This phase exists so the project does not assume:

- one specific AI vendor
- one specific agent topology
- one fixed review/implementation structure

Instead, the project starts by choosing its operating model, then the rest of the skeleton follows that choice.

Files:

- `operating-model-interview.md`
- `operating-model-options.md`
- `operating-model-record-template.md`
