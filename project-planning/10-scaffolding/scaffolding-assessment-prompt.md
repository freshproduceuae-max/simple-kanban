1. Assess readiness
2. If scaffolding is needed, create a canonical scaffolding plan
3. Keep it minimal and bounded

The scaffolding plan must include:

- objective

- exact files or areas to create or modify
- numbered implementation steps
- acceptance criteria
- risks or blockers
- what is explicitly out of scope

## Constraints

- Do not implement the scaffolding
- Do not create feature-specific implementation plans yet
- Do not start coding

## Deliverable For This Step

When finished, report back with:

1. Scaffolding assessment
2. If needed, scaffolding plan created
3. Scope boundary
4. Execution readiness
5. Proposed next step
