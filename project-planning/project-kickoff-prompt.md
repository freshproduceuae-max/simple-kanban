# Project Kickoff Prompt

Use this if you want one direct prompt instead of the shorter phrase in `COPY-INSTRUCTIONS.md`.

```text
Refer to the project planning folder and use it as the operating skeleton for this project.

Start by reading:

1. `project-planning/START-HERE.md`
2. `project-planning/README.md`

Then do the following:

1. Ask me which operating model I want to use for this project.
2. Record that operating model in the project's docs.
3. Read `project-planning/project-folder-scaffold.md` and establish canonical docs folders if needed.
4. Proceed through the phases in order.

Important:

- Do not skip phases.
- Do not start coding before the planning skeleton says it is time.
- Do not merge without a blocking review.
- Do not deploy without release tracking and rollback readiness.
```
