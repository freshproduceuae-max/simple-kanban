# Releases Folder Placeholder

Create this folder in the target project if it does not exist:

- `docs/releases/`

Purpose:

- store versioned `.rtf` release records
- preserve GitHub and Vercel traceability
- preserve rollback targets and rollback steps

Example file names:

- `release-2026-04-18-v001.rtf`
- `release-2026-04-18-auth-fix.rtf`
