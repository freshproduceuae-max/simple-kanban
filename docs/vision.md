# Product Vision: Plan

## Vision Statement

A frictionless, browser-local Kanban board that lets one person turn their scattered intentions into a visible plan in under a minute.

## Problem

Personal project planning tools have drifted toward team software: accounts, workspaces, integrations, labels, priorities, custom fields. For a single person trying to think through their own work, this overhead is noise. A sticky-note-on-a-wall simplicity is missing from the web.

## Target User

A single individual planning their own work: a student organizing coursework, a developer tracking side-project tasks, a hobbyist mapping out a weekend build. One person, one plan, one browser.

## Why This Exists / Why Now

Modern productivity tools optimize for collaboration at the expense of solo clarity. Users sign up, configure, and abandon. Plan exists because the smallest useful version of a Kanban board — three columns, draggable cards, a due date — is still the most useful version for personal planning. No one needs to build it again, but no one has shipped it cleanly either.

## Core Principles

- **Zero setup.** Open the page, start planning. No account, no onboarding.
- **Local-first.** Data lives in the browser. The user owns it; nothing leaves the device.
- **Minimal cognitive load.** Fixed columns, one card shape, one way to do things.
- **Visible urgency.** Overdue work is obvious without the user having to look for it.

## Success Looks Like

The user opens Plan, sees their board exactly as they left it, drags a card from "In Progress" to "Done," and closes the tab. No decisions about tool configuration were made. Overdue tasks caught their eye before they had to ask.

## Non-Goals

- No collaboration, sharing, or multi-user features
- No accounts, login, or cloud sync
- No customizable columns or workflows
- No labels, tags, priorities, or custom fields
- No integrations, notifications, or mobile apps
- No analytics, reporting, or history views
