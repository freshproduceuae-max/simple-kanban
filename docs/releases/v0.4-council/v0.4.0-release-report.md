## 1. What v0.4.0 is

A serial-release milestone. The browser-local v0.1 Kanban becomes a Supabase-backed single-user planner with a three-mode AI Council (Plan / Advise / Chat), persistent per-user memory, a Council Write Gate that keeps every board change behind a user tap, and a read-only session history. No multi-tenant, no billing, no task branching ‚Äî those are v0.5 / v0.6 / v1.0. This release ships **one complete voice** and **one complete safety contract**: nothing the Council says can move a card without a tap.

## 6. CD walk-through ‚Äî before flipping on

Three acceptance proxies from PRD ¬ß3 land in the release and are **not** auto-green. Each needs a human ‚Äî two need real na√Øve users, one needs a throttled Anthropic account. All three are self-contained runnable documents or unit-tested primitives ready for an end-to-end run.

### 6.1 F30 ‚Äî Anthropic 429 soft-pause under real throttle

- Mechanism: `lib/council/shared/retryOn429.ts` ‚Äî exponential backoff with 30s client-side queue cap; shelf enters a "meta-frame" soft-pause indicator on throttle; per-agent failure policy resumes after cap breach.
- Unit coverage: retry primitive + meta-frame protocol + shelf indicator are green in Vitest.
- What's deferred: the PRD ¬ß3.3 acceptance proxy specifies verification under a **deliberately throttled tier-1 Anthropic account**. That requires a real account rate-limit configuration, a recorded session artifact, and CD sign-off on user-visible behaviour during the throttle window.
- Verdict rule: one throttled session logged; meta-frame rendered; queue drained within 30s or per-agent failure path reached honestly ‚Üí GREEN.

### 6.2 F31 ‚Äî first-run 60-second stopwatch

- Doc: [`docs/releases/v0.4-council/f31-onboarding-qa-protocol.md`](./f31-onboarding-qa-protocol.md)
- What it verifies: three na√Øve users reach a tappable, trusted proposal in under 60 seconds from first sign-in, per PRD ¬ß3 acceptance proxy.
- Evidence the release captures automatically: four client-side `performance.mark` beacons ‚Äî `council:first-sign-in`, `council:first-greeting-ready`, `council:first-user-submit`, `council:first-proposal-tap`. The DevTools Console snippet in the protocol reads the marks and prints the per-user stopwatch.
- Verdict rule: all three users ‚â§ 60s ‚Üí GREEN; any single user > 60s ‚Üí tighten the path and rerun the whole three-user walk.

### 6.3 F32 ‚Äî mobile 375px sign-off

- Doc: [`docs/releases/v0.4-council/f32-mobile-375-signoff.md`](./f32-mobile-375-signoff.md)
- What it verifies: every user-tappable element on a real iPhone SE (or Chrome DevTools Device Toolbar at 375 √ó 667) clears the 44√ó44 CSS-pixel bar (design-system.md ¬ß6.2); the shelf never occludes the active board column header; the morning greeting wraps cleanly.
- Mechanisms the release already enforces: the `tap` token in `tailwind.config.ts` is the single source of truth for 44px; a unit test at `lib/__tests__/tailwind-tokens.test.ts` fails loudly if the token is deleted or retuned; six pre-existing undefined Tailwind utilities (`rounded-radius-sm`, `border-ink-40/70`, `text-ink-50/70/900`-under-old-names, `text-surface-page`) were remapped to canonical ¬ß4.2 tokens during the F32 audit.
- Surfaces requiring the walk: `/sign-in`, `/` (empty board + with-tasks + task dialog), shelf composer + ModePicker + chip input + "How I got here" reveal + memory-recall reveal, proposal card approve / try-again, `/history` (search + filters + per-row Delete + pagination), `/settings/council` (A/B/C/D cards + full-purge).

## 7. Known gaps and follow-ups

Not blockers to v0.4.0. Documented so nothing silently rots.

1. **`simple-kanban-v0-4-beta.vercel.app` alias was not set at beta cut.** The `v0.4.0-beta` tag + `release/v0.4-beta` branch preserve the git code at commit `b9ea623`; no Vercel alias was applied. A retroactive alias can still be set by identifying the preview deployment for `b9ea623` and running `npx vercel alias set <dep-url> simple-kanban-v0-4-beta.vercel.app`.
2. **Undefined `--color-ink-300` token used by `/history` and `/settings/council` delete notices.** Pre-existing borderless success-banner styling. A spawned chore (title: *Fix undefined --color-ink-300 token in delete notices*) carries the one-line remap to `border-border-default`. Does not affect tap targets.
3. **Deferred acceptance proxies (F30 throttle verification + F31 stopwatch + F32 375px walkthrough).** Authored and in-repo; require a CD on a real device / real throttled account to execute and append their result to `progress.md` via the templates in each protocol doc. This is the one remaining open front for release GREEN.
